# Rules para docs/incidents

Documentación de incidentes, postmortems y problemas resueltos.

## Formato

Cada incident report debe incluir:
- **Síntomas:** Qué se observó
- **Causa Raíz:** Por qué sucedió
- **Limpieza:** Qué se hizo para remediarlo
- **Prevención:** Cómo evitar en el futuro
- **Lecciones Aprendidas:** Qué se aprendió

## Ejemplos

- `dummy-sessions-cleanup.md` - Sesiones vacías creadas por session-save
